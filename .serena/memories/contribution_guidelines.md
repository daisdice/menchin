# Contribution Guidelines & Rules

## Language
- **JAPANESE ONLY**: All responses and plans must be in Japanese.

## Workflow (GitHub Flow)
- Create a specific branch from `main` for development.
- Branch naming convention:
    - `feature/**`
    - `fix/**`
    - `chore/**`
- Verify locally before merging to `main`.
- Merge to `main` after development is complete.
- Use `gh` command for GitHub operations.

## Code Style
- Follow `eslint` configurations.
- Use TypeScript for type safety.
